《挑战程序设计竞赛》——Expedition(POJ 2431)

优先队列
ACM题解
发布日期:   2020-02-03

原题链接

题目

原题链接

你需要驾驶一辆卡车行驶L单位距离。 最开始时，卡车上有P单位的汽油。卡车每开1单位距离需要消耗一单位汽油。如果在途中车上的汽油耗尽，卡车就无法继续前行，因而无法到达终点。在途中一共有N个加油站。第i个加油站在距离终点Ai单位距离的地方，最多可以给卡车加Bi单位的汽油。假设卡车的燃料箱的容量是无限大的，无论加多少油都没问题。那么请问卡车能否到达终点？如果可以，最少需要加多少次油？如果可以到达终点，输出最少的加油次数，否则输出-1.

1<=N<=10000

1<=L<=1000000,1<=P<=1000000

1<=Ai<=L,1<=Bi<=L

题解

思路

首先考虑忽略数据大小的情况。

加油站在途中按直线分布，位置确定。也就是说当卡车行驶到一点Xi时，可以选择在从起点到Xi间任意的加油站内加油。而为了能够尽量少的加油，我们应该在汽油尽量少后再加油。选择的加油站自然是油量最多的一家。关键是先忽略在途中加油，先考虑把油耗尽，再加油。

为了模拟这一过程，我们可以创建一个数组，在油耗尽之前，每经过一个加油站就把其位置添加进数组内。当油耗尽后，再从数组内存储的加油站中选出油量最大的一个删除。如果油耗尽时数组内没有加油站且未到达终点，则卡车不能到达终点。

但是“解题不看数据范围就是耍流氓”。本题的加油站最多可以有10000个。如果采用数组，每次选取最大值时的还需要对数组元素进行排序，这必然会导致超时。因此我们需要引入一种数据结构——优先队列。

关于优先队列的具体内容这里就略去了。

代码

#include<iostream>
#include<algorithm>
#include<queue>

using namespace std;

priority_queue <int> q;
int n, ans, i, flag, L, P, mile;
struct NODE{
    int a, b;
}node[10050];

bool cmp(NODE x, NODE y){
    return x.a<y.a;
}

int main(void)
{
    cin>>n;
    for(i=0; i<n; i++)
        cin>>node[i].a>>node[i].b;
    cin>>L>>P;
    for(i=0; i<n; i++)
        node[i].a=L-node[i].a;
    sort(node, node+i, cmp);
    i=0;
    while(node[i].a<=P&&i<n)//防越界
    {
        q.push(node[i].b);
        i++;
    }  
    while(P<L)
    {   
        if(q.empty())
        {
            flag=1;
            break;
        }
        else
        {
            P+=q.top();
            q.pop();
            ans++;
            while(node[i].a<=P&&i<n)//防越界
            {
                q.push(node[i].b);
                i++;
            } 
        }
    }
    if(!flag)
        cout<<ans<<endl;
    else
        cout<<-1<<endl;
    return 0;
}

这里我采用结构而非两个数组存储加油站信息。因为后续操作需要将加油站按据起点距离进行排序，采用第二种方法会使距离和加油量两组数据不匹配。

总结

  1. 读题很重要。一开始题目的意思理解错了，把问题想得过于复杂。

  2. 注意数组边界。出现用于数组计数的i时最好都加上限制条件，天知道会碰上什么坑爹的样例。
